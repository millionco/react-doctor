# Project Structure & Configuration

## Recommended file layout

```
├── app.config.ts            # TanStack Start configuration
├── vite.config.ts           # Vite configuration (if using Vite plugin)
├── tsconfig.json
├── package.json
└── src/
    ├── start.ts             # Global middleware (optional, not in default template)
    ├── router.tsx            # Router instance configuration
    ├── routeTree.gen.ts     # Auto-generated route tree — never edit
    ├── routes/
    │   ├── __root.tsx       # Root layout
    │   ├── index.tsx        # Home page (/)
    │   ├── posts.tsx        # /posts layout
    │   ├── posts.index.tsx  # /posts exact
    │   └── posts.$postId.tsx
    ├── components/          # Shared React components
    ├── utils/
    │   ├── users.functions.ts   # Server function wrappers
    │   ├── users.server.ts      # Server-only logic
    │   └── schemas.ts           # Shared validation schemas
    ├── middleware/
    │   └── auth.ts          # Auth middleware
    └── styles/
        └── globals.css
```

## app.config.ts

The entry point for TanStack Start configuration:

```tsx
import { defineConfig } from '@tanstack/start/config'

export default defineConfig({
  server: {
    preset: 'node-server',
  },
})
```

### Deployment presets

| Preset | Target |
|---|---|
| `node-server` | Node.js / Docker |
| `vercel` | Vercel |
| `netlify` | Netlify |
| `cloudflare-workers` | Cloudflare Workers |
| `railway` | Railway |
| `bun` | Bun |
| `nitro` | Generic Nitro |

### Vite plugin configuration

```tsx
// vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { tanstackStart } from '@tanstack/react-start/plugin/vite'

export default defineConfig({
  plugins: [
    tanstackStart(),
    react(),
  ],
})
```

## Router configuration

```tsx
// src/router.tsx
import { createRouter } from '@tanstack/react-router'
import { routeTree } from './routeTree.gen'

export const createAppRouter = () =>
  createRouter({
    routeTree,
    defaultPreload: 'intent',
  })

declare module '@tanstack/react-router' {
  interface Register {
    router: ReturnType<typeof createAppRouter>
  }
}
```

The `Register` interface enables global type inference for all routes, links, and navigation calls.

## src/start.ts (global middleware)

Not included in the default template — create it when you need global middleware:

```tsx
import { createStart } from '@tanstack/react-start'
import { authMiddleware } from './middleware/auth'

export const startInstance = createStart(() => ({
  requestMiddleware: [authMiddleware],
}))
```

## routeTree.gen.ts

Auto-generated by the TanStack Router plugin. Never edit this file manually. Add it to `.gitignore` or commit it — both approaches work, but committing is recommended for CI reproducibility.

## Path aliases

Configure `@/` or `~/` aliases for clean imports:

```json
// tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "~/*": ["./src/*"]
    }
  }
}
```

Use with `vite-tsconfig-paths` plugin:

```tsx
// vite.config.ts
import tsconfigPaths from 'vite-tsconfig-paths'

export default defineConfig({
  plugins: [
    tsconfigPaths(),
    tanstackStart(),
    react(),
  ],
})
```

## Rules

- **DO** keep routes in `src/routes/` — the router plugin watches this directory.
- **DO** use path aliases (`~/`) for imports deeper than two levels.
- **DO** register the router type globally via the `Register` interface.
- **DO** set `defaultPreload: 'intent'` on the router for better perceived performance.
- **DO NOT** edit `routeTree.gen.ts` manually.
- **DO NOT** put business logic in route files — extract to `utils/` and `middleware/`.
- **DO NOT** mix server-only imports (`.server.ts`) in client-accessible files.
